校园二手物品交易信息

  • author: shiwenbo <63091158@qq.com>
  • version：v1.0

总览

环境

  • CodeIgniter 3.1.0
  • php >= 5.6.30
  • mysql >= 5.7.18
  • php扩展: mbstring, curl
  • 并没有用腾讯云的SDK. wafer相关配置，COS相关配置一律不用管

后端文件结构

  • application/controllers 控制器
  • application/models 模型
  • application/config 配置文件
  • index.php 入口文件(在此已配置自动切换环境，无需手动操作)
  • public 静态资源文件
  • uploads 上传文件仓库
  • uploads/pannel.php 查看服务器文件库(仅在测试环境可用)
  • system 系统核心文件 (不要乱动)
  • vendor (tx的东西不知道是啥，不要乱动)

维护相关

  • api.weixin.qq.com 的DNS目前是写死的，如果 login 接口无法使用请检查DNS是否正确

开始使用

本项目分为测试环境和生产环境

  • 测试环境 base_url 为 https://ntfxnyrz.qcloud.la
  • 生产环境 base_url 为 https://xjtuershou.tiaozhan.com/api/v1.0

请求和响应格式

例如：以GET方式访问welcome接口：

GET {base_url}/welcome

Response 200

  • 成功响应：
    {
        "code":0,
        "data":
        {
            "msg": "Hello World"
        }
    }
  • 错误响应
    {
        "code":-1,
        "error":"错误信息"
    }

状态码 code

code 含义
0 正常
-1 登录状态错误
-2 表单参数错误
-3 安全性错误
-4 文件上传错误
-5 netid未授权，操作失败

上传模块

上传文件 POST {base_url}/upload

  • 文件大小：图片：0.5M 视频: 20M
  • 支持格式：jpg|mov|mp4

Request (multipart/form-data)

参数 类型 可选参数 注释
messageid text
openid text
token text
type text 'img' 或 'video'
no text 为防止前端上传时异步产生乱序，在这里规定文件顺序
file file

Response 200

    {
        "code": 0,
        "data": 
        {
            "filename": "58-1526733386.jpg"
        }
    }

获取资源 GET {base_url}/uploads/{filename}

例如: 以 GET 方式请求 https://ntfxnyrz.qcloud.la/uploads/58-1526733386.jpg 会接收到图片资源
